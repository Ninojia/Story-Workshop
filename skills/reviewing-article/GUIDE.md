# 文章审校

## 概述

对完整草稿进行系统化的三轮审校，每轮专注一个维度。发现的问题按严重程度分级处理，最终产出可发布的终稿。

**进入条件：** 已有完整草稿（来自内容执行阶段）

**产出文件：**
- 审校报告：`docs/writing/YYYY-MM-DD-<topic>-review.md`
- 终稿：`docs/writing/YYYY-MM-DD-<topic>-final.md`

## 审校流程

```
读取完整草稿
    ↓
第1轮：内容审校（信息准确？逻辑通顺？论据充分？）
    ↓
第2轮：表达审校（文字流畅？风格统一？节奏得当？）
    ↓
第3轮：规范审校（格式正确？用语规范？数据核实？）
    ↓
生成审校报告 → 呈现给用户
    ↓
应用修复 → 保存终稿
```

## 第1轮：内容审校

**核心问题：文章说的对不对？说的全不全？**

### 检查清单

**逻辑维度：**
- [ ] 核心信息是否在前1/3就明确传达？
- [ ] 每个章节是否都在支撑核心信息？
- [ ] 论证链条是否完整（没有跳跃式推理）？
- [ ] 各章节之间是否有矛盾或重复？
- [ ] 结论是否从前文自然推导出来？

**事实维度：**
- [ ] 数据是否准确（与素材一致）？
- [ ] 人物引用是否准确（没有篡改原意）？
- [ ] 时间线是否正确？
- [ ] 技术描述是否准确（不过度简化也不错误）？

**完整性维度：**
- [ ] 写作简报中的关键素材是否都用到了？
- [ ] 有没有空洞的断言缺乏支撑？
- [ ] 读者看完会不会有"然后呢？"的困惑？

### 问题记录格式

```markdown
### 第1轮：内容审校

**[严重]** 第3节，第2段
- 问题：数据引用有误
- 当前文本："成功率从95%提升到99.9%"
- 素材原文："从97%提升到99.9%"
- 修复：更正为97%

**[中等]** 第4节
- 问题：缺少关键论据支撑
- 当前文本："优化效果非常显著"
- 分析：没有给出具体数据
- 修复：补充"小文件传输从518秒降到8秒"

**[轻微]** 第2节，第1段
- 问题：时间线描述模糊
- 当前文本："后来团队决定..."
- 修复：补充具体时间"2024年12月底，团队决定..."

**亮点：**
- 开头场景切入非常有效，一下抓住读者
- 第3节的技术类比很到位
```

## 第2轮：表达审校

**核心问题：文章读起来爽不爽？**

### 检查清单

**流畅度：**
- [ ] 章节间过渡是否自然（没有突兀的跳转）？
- [ ] 段落间是否有清晰的逻辑连接？
- [ ] 有没有"读到一半要回去看前面"的地方？

**节奏感：**
- [ ] 是否有快慢交替的节奏变化？
- [ ] 重要内容是否有足够的展开空间？
- [ ] 有没有可以删掉而不影响理解的段落？

**文字质量：**
- [ ] 句式是否有变化（不是全部长句或全部短句）？
- [ ] 有没有拗口的句子（读出来不顺）？
- [ ] 修饰词是否克制（"非常"/"极其"出现次数）？
- [ ] 是否存在"正确但无聊"的句子（可以说得更有意思）？

**风格一致性（对标「冷静的温度」风格体系）：**
- [ ] 全文语气是否一致（没有忽然正式忽然口语）？
- [ ] 人称视角是否一致？
- [ ] 文章类型的风格特征是否贯穿始终？
- [ ] 开头是否一句话亮矛盾，没有背景铺垫？
- [ ] 人物是否靠行动+引语立住，没有贴标签或心理描写？
- [ ] 引语是否删掉了无效语气词，每句携带信息？
- [ ] 数据是否嵌入叙事，有 before vs after 对比？
- [ ] 比喻是否少而精（一概念一比喻），生活化但利落？
- [ ] 是否存在感叹号或空洞情绪词？
- [ ] 结尾是否落在"未完成"上，避免了口号式总结？

**金句密度：**
- [ ] 每500-800字是否至少有一个值得划线的句子（2000字文章需3-4个金句）？
- [ ] 金句是否放在了有力的位置（段首、段尾、独立成行）？

### 常见的表达问题

| 问题类型 | 示例 | 改进方向 |
|---------|------|---------|
| **空话套话** | "经过团队的不懈努力..." | 删掉，或换成具体细节 |
| **被动式堆砌** | "问题被发现了，方案被制定了，结果被验证了" | 改为主动语态 |
| **抽象概括** | "用户体验得到了显著提升" | 换成具体场景或数据 |
| **过度修饰** | "这是一个非常了不起的、极其重要的创新" | 去掉修饰词，让事实说话 |
| **报告体** | "综上所述，本项目取得了以下成果..." | 换成自然表达 |
| **平铺直叙** | "首先做了A，然后做了B，最后做了C" | 加入转折、悬念、细节 |
| **开头无聊** | "随着互联网的发展..." | 从矛盾/张力切入，一句话亮刀 |
| **结尾八股** | "展望未来，我们将继续努力..." | 落在"未完成"上，用细节/引语收束 |
| **贴标签** | "他是一个追求极致的技术人" | 用具体动作和引语让人物自己说话 |
| **数据孤岛** | 单独列数据清单 | 嵌入叙事，配 before vs after 对比 |
| **比喻堆砌** | "像建筑师搭建智慧大厦..." | 一个概念最多一个比喻，利落表达 |
| **感叹号滥用** | "这远超预期！太厉害了！" | 删掉感叹号，用事实本身传递力量 |

### 问题记录格式

```markdown
### 第2轮：表达审校

**[中等]** 第1节 → 第2节 过渡
- 问题：转换生硬
- 当前："...项目就这样开始了。QQ闪传的前身是..."
- 分析：缺少自然的衔接
- 修复建议：用一个承上启下的句子连接

**[轻微]** 全文
- 问题："非常"出现7次
- 修复：保留2次最必要的，其余删除或换词

**[轻微]** 第5节
- 问题：结尾落入"总结+展望"的套路
- 当前："未来，团队将继续在..."
- 修复建议：用一个小细节或人物金句收束

**节奏评估：**
- ✅ 第1-3节节奏很好，快慢交替
- ⚠️ 第4-5节整体偏平，缺少情感起伏
- 建议：在第4节中间插入一个小故事/细节来打破平淡

**金句盘点：**
- ✅ 第1节："每一个用户的声音，都值得被认真对待。" — 很好
- ⚠️ 第3-4节缺少金句，建议从素材金句中选取补充
```

## 第3轮：规范审校

**核心问题：文章的格式和细节有没有问题？**

### 检查清单

**排版格式：**
- [ ] 小标题层级是否正确且一致？
- [ ] 加粗/强调是否使用得当（不滥用）？
- [ ] 列表格式是否统一？
- [ ] 段落间距是否一致？

**中文规范：**
- [ ] 中英文之间是否有空格？（`QQ 闪传` 而非 `QQ闪传`）
- [ ] 标点符号是否正确？（中文标点 vs 英文标点）
- [ ] 数字格式是否一致？
- [ ] 破折号是否使用 `——`？

**数据核实：**
- [ ] 所有数字是否与素材一致？
- [ ] 百分比计算是否正确？
- [ ] 时间/日期是否准确？

**其他：**
- [ ] 有无错别字/笔误？
- [ ] 专有名词拼写是否正确？
- [ ] 字数是否在目标范围内？

### 问题记录格式

```markdown
### 第3轮：规范审校

**格式问题：**
- 第2节小标题用了三级标题 `###`，其他节用的二级 `##` → 统一为 `##`
- 第4节有一处英文逗号 `,` → 改为中文逗号 `，`

**数据核实：**
- ✅ 月活数据 1180 万 — 与素材一致
- ✅ 成功率 99.9% — 与素材一致
- ⚠️ "20多人的团队" — 素材中说"约20人"，建议统一

**字数统计：**
- 当前：XXXX 字
- 目标：XXXX 字（默认约 2000 字）
- 偏差：[百分比]
- 处理：[可接受（±15%内） / 需精简 / 需扩展]
```

## 问题严重程度分级

| 等级 | 定义 | 处理方式 |
|------|------|---------|
| **🔴 严重** | 事实错误、逻辑漏洞、重大遗漏 | 逐一展示，提出修复方案，等待用户确认 |
| **🟡 中等** | 表达不佳、衔接生硬、风格偏差 | 打包展示，统一修复 |
| **🟢 轻微** | 格式问题、小瑕疵、微调优化 | 直接修复，在报告中记录 |

## 应用修复

### 修复流程

1. **呈现审校报告摘要给用户**
   - 总体评价
   - 严重问题列表
   - 主要改进建议

2. **处理严重问题**
   - 逐一展示：当前文本 → 问题分析 → 修复方案
   - 等待用户确认每一处修复

3. **批量处理中等/轻微问题**
   - 告知用户"以下问题我一并修复了"
   - 列出修改清单

4. **生成终稿**

### 修复后的二次检查

修复完成后，快速通读全文确认：
- [ ] 严重问题已全部修复
- [ ] 修复没有引入新的问题
- [ ] 文章整体仍然流畅连贯
- [ ] 字数在合理范围内

## 核心原则

- **三轮分离：** 每轮只关注一个维度，不混合检查。先确保内容对，再追求表达好，最后处理格式。
- **严重问题不放过：** 事实错误、逻辑漏洞必须用户确认后修复。
- **保留亮点：** 审校的目的是改掉坏的，不是改掉好的。标记亮点，提醒不要误改。
- **修复不引入新问题：** 每次修复后检查是否影响了上下文。
- **适可而止：** 不存在完美的文章。当问题只剩"这个逗号用不用"的级别时，就该收手了。
