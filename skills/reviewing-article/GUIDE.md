# 文章审校

## 概述

对完整草稿进行系统化的三轮审校，每轮专注一个维度。发现的问题按严重程度分级处理，最终产出可发布的终稿。

**进入条件：** 已有完整草稿（来自内容执行阶段）

**产出文件：**
- 审校报告：`docs/writing/YYYY-MM-DD-<topic>-review.md`
- 终稿：`docs/writing/YYYY-MM-DD-<topic>-final.md`

## 审校流程

```
读取完整草稿
    ↓
第1轮：内容审校（信息准确？逻辑通顺？论据充分？）
    ↓
第2轮：表达审校（文字流畅？风格统一？节奏得当？）
    ↓
第3轮：规范审校（格式正确？用语规范？数据核实？）
    ↓
生成审校报告 → 呈现给用户
    ↓
应用修复 → 保存终稿
```

## 第1轮：内容审校

**核心问题：文章说的对不对？说的全不全？**

### 检查清单

**逻辑维度：**
- [ ] 核心信息是否在前1/3就明确传达？
- [ ] 每个章节是否都在支撑核心信息？
- [ ] 论证链条是否完整（没有跳跃式推理）？
- [ ] 各章节之间是否有矛盾或重复？
- [ ] 结论是否从前文自然推导出来？

**事实维度：**
- [ ] 数据是否准确（与素材一致）？
- [ ] 人物引用是否准确（没有篡改原意）？
- [ ] 时间线是否正确？
- [ ] 技术描述是否准确（不过度简化也不错误）？

**完整性维度：**
- [ ] 写作简报中的关键素材是否都用到了？
- [ ] 有没有空洞的断言缺乏支撑？
- [ ] 读者看完会不会有"然后呢？"的困惑？

### 问题记录格式

```markdown
### 第1轮：内容审校

**[严重]** 第3节，第2段
- 问题：数据引用有误
- 当前文本："成功率从95%提升到99.9%"
- 素材原文："从97%提升到99.9%"
- 修复：更正为97%

**[中等]** 第4节
- 问题：缺少关键论据支撑
- 当前文本："优化效果非常显著"
- 分析：没有给出具体数据
- 修复：补充"小文件传输从518秒降到8秒"

**[轻微]** 第2节，第1段
- 问题：时间线描述模糊
- 当前文本："后来团队决定..."
- 修复：补充具体时间"2024年12月底，团队决定..."

**亮点：**
- 开头场景切入非常有效，一下抓住读者
- 第3节的技术类比很到位
```

## 第2轮：表达审校

**核心问题：文章读起来爽不爽？**

### 检查清单

**流畅度：**
- [ ] 章节间过渡是否自然（没有突兀的跳转）？
- [ ] 段落间是否有清晰的逻辑连接？
- [ ] 有没有"读到一半要回去看前面"的地方？

**节奏感：**
- [ ] 是否有快慢交替的节奏变化？
- [ ] 重要内容是否有足够的展开空间？
- [ ] 有没有可以删掉而不影响理解的段落？

**文字质量：**
- [ ] 句式是否有变化（不是全部长句或全部短句）？
- [ ] 有没有拗口的句子（读出来不顺）？
- [ ] 修饰词是否克制（"非常"/"极其"出现次数）？
- [ ] 是否存在"正确但无聊"的句子（可以说得更有意思）？

**风格一致性（对标「冷静的温度」风格体系）：**
- [ ] 全文语气是否一致（没有忽然正式忽然口语）？
- [ ] 人称视角是否一致？
- [ ] 文章类型的风格特征是否贯穿始终？
- [ ] 开头是否一句话亮矛盾，没有背景铺垫？
- [ ] 人物是否靠行动+引语立住，没有贴标签或心理描写？
- [ ] 引语是否删掉了无效语气词，每句携带信息？
- [ ] 数据是否嵌入叙事，有 before vs after 对比？
- [ ] 比喻是否少而精（一概念一比喻），生活化但利落？
- [ ] 是否存在感叹号或空洞情绪词？
- [ ] 结尾是否落在"未完成"上，避免了口号式总结？

**🔴 AI 味检测（准则十一，必查）：**

这是表达审校中**最重要的检查项**。AI 写作最大的破绽不是单句差，而是全文过于整齐、正向、完成度太高。逐项检查：

*结构层面：*
- [ ] 把每章骨架抽象为一句话——是否超过半数结构相似？（如都是"以前→AI→提效→更有价值"）
- [ ] 章节小标题格式是否过于统一？（应有引语型、动作型、名字型等混合）
- [ ] 结尾是否使用了"第一……第二……第三……"式教科书收束？

*修辞层面：*
- [ ] 金句式总结句（"这正是……""这可能是最大的……"）全文是否超过2处？
- [ ] 以下模板句式全文是否各超过1次："不是……而是……""每个人的边界……""一场……的革命"？
- [ ] 排比/并列结构（3个以上）是否都已用不同节奏的句子打散？
- [ ] 连续5段以上是否每句都很"顺"？——真人写作会有磕绊、犹豫、不完美的停顿

*内容层面：*
- [ ] 全文正面/负面内容比例是否在 7:3 左右？（不能全是正向收益）
- [ ] 是否有至少2-3处真实的代价、失败、不确定描述？（"中间卡了两天""差点放弃""说实话现在也没想清楚"）
- [ ] "人类犹豫感"是否足够？——不是编造的假犹豫，而是从素材中提取的真实不确定

**AI 味判定标准：**
- 以上检查项中有 **3项以上** 未通过 → 标记为 **🔴 严重：AI 味过重**，必须系统性重构
- 有 **1-2项** 未通过 → 标记为 **🟡 中等**，针对性修复
- 全部通过 → ✅ 自然度良好

**金句密度：**
- [ ] 每500-800字是否至少有一个值得划线的句子（2000字文章需3-4个金句）？
- [ ] 金句是否放在了有力的位置（段首、段尾、独立成行）？

### 常见的表达问题

| 问题类型 | 示例 | 改进方向 |
|---------|------|---------|
| **空话套话** | "经过团队的不懈努力..." | 删掉，或换成具体细节 |
| **被动式堆砌** | "问题被发现了，方案被制定了，结果被验证了" | 改为主动语态 |
| **抽象概括** | "用户体验得到了显著提升" | 换成具体场景或数据 |
| **过度修饰** | "这是一个非常了不起的、极其重要的创新" | 去掉修饰词，让事实说话 |
| **报告体** | "综上所述，本项目取得了以下成果..." | 换成自然表达 |
| **平铺直叙** | "首先做了A，然后做了B，最后做了C" | 加入转折、悬念、细节 |
| **开头无聊** | "随着互联网的发展..." | 从矛盾/张力切入，一句话亮刀 |
| **结尾八股** | "展望未来，我们将继续努力..." | 落在"未完成"上，用细节/引语收束 |
| **贴标签** | "他是一个追求极致的技术人" | 用具体动作和引语让人物自己说话 |
| **数据孤岛** | 单独列数据清单 | 嵌入叙事，配 before vs after 对比 |
| **比喻堆砌** | "像建筑师搭建智慧大厦..." | 一个概念最多一个比喻，利落表达 |
| **感叹号滥用** | "这远超预期！太厉害了！" | 删掉感叹号，用事实本身传递力量 |
| **章节骨架重复** | 每章都是"以前→AI→提效→价值" | 不同章节用不同叙事重心和结构 |
| **金句过密** | 每章末尾都来一句升华 | 全文金句总结句≤2处，其余用事实收束 |
| **模板句式** | "这正是……""不是A而是B" | 同一句式全文不超过1次 |
| **全员正向** | 9个人都提效、都成功 | 加入代价、失败、不确定，正负比7:3 |
| **缺乏犹豫感** | 每句话都很顺，无停顿 | 加入真实的犹豫和不确定表达 |
| **排比过齐** | 三个"不再……了" | 中间插入不同节奏的句子打散 |

### 问题记录格式

```markdown
### 第2轮：表达审校

**[中等]** 第1节 → 第2节 过渡
- 问题：转换生硬
- 当前："...项目就这样开始了。QQ闪传的前身是..."
- 分析：缺少自然的衔接
- 修复建议：用一个承上启下的句子连接

**[轻微]** 全文
- 问题："非常"出现7次
- 修复：保留2次最必要的，其余删除或换词

**[轻微]** 第5节
- 问题：结尾落入"总结+展望"的套路
- 当前："未来，团队将继续在..."
- 修复建议：用一个小细节或人物金句收束

**节奏评估：**
- ✅ 第1-3节节奏很好，快慢交替
- ⚠️ 第4-5节整体偏平，缺少情感起伏
- 建议：在第4节中间插入一个小故事/细节来打破平淡

**金句盘点：**
- ✅ 第1节："每一个用户的声音，都值得被认真对待。" — 很好
- ⚠️ 第3-4节缺少金句，建议从素材金句中选取补充
```

## 第3轮：规范审校

**核心问题：文章的格式和细节有没有问题？**

### 检查清单

**排版格式：**
- [ ] 小标题层级是否正确且一致？
- [ ] 加粗/强调是否使用得当（不滥用）？
- [ ] 列表格式是否统一？
- [ ] 段落间距是否一致？

**中文规范：**
- [ ] 中英文之间是否有空格？（`QQ 闪传` 而非 `QQ闪传`）
- [ ] 标点符号是否正确？（中文标点 vs 英文标点）
- [ ] 数字格式是否一致？
- [ ] 破折号是否使用 `——`？

**数据核实：**
- [ ] 所有数字是否与素材一致？
- [ ] 百分比计算是否正确？
- [ ] 时间/日期是否准确？

**其他：**
- [ ] 有无错别字/笔误？
- [ ] 专有名词拼写是否正确？
- [ ] 字数是否在目标范围内？

### 问题记录格式

```markdown
### 第3轮：规范审校

**格式问题：**
- 第2节小标题用了三级标题 `###`，其他节用的二级 `##` → 统一为 `##`
- 第4节有一处英文逗号 `,` → 改为中文逗号 `，`

**数据核实：**
- ✅ 月活数据 1180 万 — 与素材一致
- ✅ 成功率 99.9% — 与素材一致
- ⚠️ "20多人的团队" — 素材中说"约20人"，建议统一

**字数统计：**
- 当前：XXXX 字
- 目标：XXXX 字（默认约 2000 字）
- 偏差：[百分比]
- 处理：[可接受（±15%内） / 需精简 / 需扩展]
```

## 第4轮：发布口径风险审校

**核心问题：这篇文章发出去之后，有没有可能被截图、被追问、或者引发不必要的关注？**

这一轮不判断文章写得好不好，只判断**发出去安不安全**。

### 风险等级定义

| 等级 | 定义 | 处理方式 |
|------|------|---------|
| **🔴 P0（必须改）** | 发出去后大概率引发负面关注或内部追问 | 强制修改，提供改写建议 |
| **🟡 P1（建议改）** | 存在被断章取义或产生歧义的风险 | 提示用户确认，给出改写选项 |
| **🟢 P2（可保留）** | 风险可控，有事实/证据支撑 | 标注说明，不强制改 |

### 检查清单

**具名与角色风险：**
- [ ] 是否出现了真实人名（全名或足以识别的称谓）？
  - 内网文章也需确认当事人是否同意具名
  - 领导/高管具名 + 具体决策行为 → 升为 P0
- [ ] 是否出现了外部公司/个人（KOC、竞品、合作方）的具体行为描述？

**争议触发句：**
- [ ] 是否有可被断章取义的单句（截出来脱离语境会引发误读）？
  - ❌ "波峰时稍微控制速度" → 截图后变成"QQ承认故意限速"
  - ✅ "当前并未对用户限速，已储备动态调整手段"
- [ ] 是否有暗示"曾经出过事"或"内部有争议"的敏感表述？

**绝对化与自夸表述：**
- [ ] 是否出现 `业界首创`、`唯一`、`最骄傲`、`颠覆性`？
  - 建议替换为：`创新尝试`、`阶段性探索`、`具有代表性的实践`
- [ ] 是否有明确的竞品对比，且措辞带有打压意味？

**目标数字与承诺性表述：**
- [ ] 是否出现了未经公开确认的具体目标值（如 H1 目标 2000 万）？
  - 建议改为趋势描述：`继续保持增长势头`、`向更大规模迈进`
- [ ] 是否有听起来像"公开承诺"的表述（会被截图当做公司官方表态）？

**合规/法律风险词：**
- [ ] 是否出现了涉及保密要求、未发布功能、内部决策细节的描述？
- [ ] 是否提到了安全策略、反作弊、黑产应对的具体手段？（易成为攻击面）

### 改写参考

| AI 版原句（高风险） | 人工版改写（低风险） | 风险类型 |
|-------------------|-------------------|---------|
| `KOC体验完绷不住了，直接发了条朋友圈` | `内测用户收到了大量积极的反馈` | P0 - 戏剧化+外部具名 |
| `有保密要求，不能公开说` | （删除） | P0 - 暴露保密机制 |
| `波峰时稍微控制速度` | `当前并未限速，已储备动态调整手段` | P0 - 可被截图断章 |
| `业界首创` | `创新尝试` | P1 - 绝对化自夸 |
| `最骄傲的技术突破之一` | `具有代表性的创新实践` | P1 - 自夸 |
| `H1目标是2000万，有信心` | （删除具体数字，保留信心表述） | P1 - 公开承诺 |
| `校长直接把截图转到项目群` | `截图被转到了项目群` | P1 - 领导具名+具体行为 |
| `嘉伟说……`（高频具名） | `团队认为……` | P1 - 去具名化 |

```markdown

**🟡 P1（建议改）：**
```

## 问题严重程度分级

| 等级 | 定义 | 处理方式 |
|------|------|---------|
| **🔴 严重** | 事实错误、逻辑漏洞、重大遗漏 | 逐一展示，提出修复方案，等待用户确认 |
| **🟡 中等** | 表达不佳、衔接生硬、风格偏差 | 打包展示，统一修复 |
| **🟢 轻微** | 格式问题、小瑕疵、微调优化 | 直接修复，在报告中记录 |

## 应用修复

### 修复流程

1. **呈现审校报告摘要给用户**
   - 总体评价
   - 严重问题列表
   - 主要改进建议

2. **处理严重问题**
   - 逐一展示：当前文本 → 问题分析 → 修复方案
   - 等待用户确认每一处修复

3. **批量处理中等/轻微问题**
   - 告知用户"以下问题我一并修复了"
   - 列出修改清单

4. **生成终稿**

### 修复后的二次检查

修复完成后，快速通读全文确认：
- [ ] 严重问题已全部修复
- [ ] 修复没有引入新的问题
- [ ] 文章整体仍然流畅连贯
- [ ] 字数在合理范围内

## 核心原则

- **三轮分离：** 每轮只关注一个维度，不混合检查。先确保内容对，再追求表达好，最后处理格式。
- **严重问题不放过：** 事实错误、逻辑漏洞必须用户确认后修复。
- **保留亮点：** 审校的目的是改掉坏的，不是改掉好的。标记亮点，提醒不要误改。
- **修复不引入新问题：** 每次修复后检查是否影响了上下文。
- **适可而止：** 不存在完美的文章。当问题只剩"这个逗号用不用"的级别时，就该收手了。
