# 内容执行

## 概述

按照写作简报（快速模式）或写作计划（完整模式）撰写文章内容。逐节完成，每节自检，定期设置人工检查点。

**进入条件：**
- 快速模式：已有用户确认的写作简报
- 完整模式：已有用户确认的写作计划

**产出文件：**
- 章节草稿：`docs/writing/drafts/YYYY-MM-DD-<topic>/section-N-<title>.md`
- 完整草稿：`docs/writing/YYYY-MM-DD-<topic>-draft.md`

## 执行流程

### 快速模式

```
读取写作简报 → 根据骨架一气呵成写完全文
    ↓
整体自检 → 保存草稿
    ↓
调用「文章审校」
```

快速模式下，不分节写作，直接产出完整草稿。但仍需遵循本文档的写作规范。

### 完整模式

```
读取写作计划 → 写第N节 → 自检
    ↓
自检通过？ ──否──→ 修改
    ↓是
保存章节草稿
    ↓
本批完成（2-3节）？ ──否──→ 写下一节
    ↓是
🔴 人工检查点
    ↓
所有章节完成？ ──否──→ 继续下一批
    ↓是
合并为完整草稿 → 调用「文章审校」
```

## 写作规范

### 中文行文规范

**必须遵守：**
- 中文与英文/数字之间加空格：`QQ 闪传`、`月活 1180 万`
- 使用中文标点：句号`。` 逗号`，` 顿号`、` 引号`""`
- 数字使用阿拉伯数字：`20人` 而非 `二十人`（特定语境如"第一性原理"除外）
- 百分比用 `%` 不用 `％`
- 破折号使用 `——`（两个 em dash）

**语言风格（遵循「冷静的温度」风格体系，详见主 SKILL.md）：**
- 不用"笔者"，不用"我们认为"这类学术腔
- 不用"综上所述"、"总而言之"这类套话
- 人物引用保持口语化原貌，但删掉无效语气词，每句引语必须携带信息或态度
- 技术术语首次出现时用通俗语言解释
- 语气冷静克制，不用感叹号，不用"令人震惊""不可思议"等空洞情绪词
- 态度通过选材和措辞暗示，不直说
- 比喻少而精，一个概念最多一个比喻，选生活化场景
- 数据嵌入叙事中，配 before vs after 对比，不单独列清单

### 按文章类型的写作要领

#### 📖 人物/团队故事

**叙事技巧：**
- **场景先行：** 每个重要情节都从一个具体场景开始写。不是"团队遇到了困难"，而是"2025年7月的一天，一条小红书帖子被转进了项目群里。"
- **对话驱动：** 大量使用人物直接引语。受访者的原话比改写更真实、更有力。
- **细节为王：** "从518秒优化到8秒" 比 "性能大幅提升" 有力一万倍。
- **情感不煽情：** 写人物的真实反应，不要替读者感动。"那一刻我是被惊到的" 比 "这令人无比振奋" 更好。
- **时间锚点：** 给每个关键事件标注时间，让读者有时间感。

**节奏控制：**
- 快节奏板块（密集事件/数据）和慢节奏板块（场景/情感）交替出现
- 重要段落前后适当留白
- 金句独立成行，增强冲击力

#### 📊 项目复盘

**写作要领：**
- **结论前置：** 每个板块先给结论，再展开过程。读者不想猜。
- **决策还原：** 不只说"我们选了方案A"，要说清"当时有ABC三个选项，我们因为XX选了A"。
- **数据为骨：** 每个关键论点都要有数据支撑。优化前后的对比是最有力的叙述。
- **坑要具体：** "踩了很多坑"没有价值。"用户息屏后网络断连导致传输中断"才有价值。
- **方法论提炼：** 结尾的经验总结要简洁到可以直接抄走用。

#### 🔧 技术分享

**写作要领：**
- **类比先行：** 用生活化的类比开路。"小文件传输慢，就像送100瓶水，每次只装一瓶来回跑100趟"。
- **问题驱动：** 先讲清楚问题为什么难，读者才会关心你的方案。
- **适度简化：** 面向非本领域的技术同事写。省略实现琐碎，保留核心思路。
- **图表辅助：** 复杂架构用文字描述 + 建议配图说明。
- **效果用数据说话：** 不说"显著提升"，说"从518秒降到8秒"。

#### 🚀 产品介绍

**写作要领：**
- **用户视角：** 全程站在用户角度写。不是"我们做了什么"，是"你可以怎么用"。
- **痛点共鸣：** 开头的痛点描述要让读者感同身受。
- **亮点聚焦：** 不贪多，选3-5个最核心的亮点展开，其他一笔带过。
- **对比出价值：** "原来需要4-5步，现在只需要发送+下载" 比单独说"操作简便"有力。
- **行动引导：** 告诉读者怎么用、在哪里用。

## 每节写作流程

### 1. 读取章节要求

- 回顾本节定位、核心要点、素材分配
- 检查上一节的结尾，确保衔接自然
- 确认字数目标和语气要求

### 2. 撰写内容

- 按写作计划的素材分配来写
- 从上一节的衔接点自然展开
- 关键数据和引用必须完整准确
- 收束时为下一节做铺垫

### 3. 自检清单

每节写完后检查：

**内容维度：**
- [ ] 核心要点是否都覆盖了？
- [ ] 数据/引用是否准确？
- [ ] 有没有空洞的断言（缺乏数据或例子支撑）？

**表达维度：**
- [ ] 每段不超过4句 / 120字？
- [ ] 有没有"非常""极其"等多余修饰词？
- [ ] 人物引用是否保持了口语化原貌？
- [ ] 句式有变化，不是全部长句或全部短句？

**结构维度：**
- [ ] 与上一节衔接自然？
- [ ] 为下一节做好了铺垫？
- [ ] 字数在目标的 ±15% 以内？

**风格维度（「冷静的温度」检查）：**
- [ ] 符合本文章类型的写作要领？
- [ ] 语气与全文一致？
- [ ] 开头是否一句话亮矛盾/张力？
- [ ] 人物是否靠行动+引语立住，没有贴标签或写心理？
- [ ] 数据是否嵌入叙事（非单独列表），有 before vs after 对比？
- [ ] 比喻是否少而精，生活化但不堆砌？
- [ ] 是否有感叹号或空洞情绪词需要删除？
- [ ] 结尾是否落在"未完成"上，而非口号式总结？

**反模板感检查（准则十一）：**
- [ ] 本节的叙事骨架是否和上一节不同？（不能都是"以前→AI→提效→更有价值"）
- [ ] 本节末尾是否避免了金句式升华？（全文金句总结句不超过2处）
- [ ] 是否包含至少一处真实的犹豫/不确定/代价描述？
- [ ] 是否有"这正是……""不是……而是……"等模板句式？如有，全文是否已超过1处？
- [ ] 排比/并列句式是否超过3个？如有，是否已用不同节奏的句子打散？

### 4. 保存章节草稿

保存至 `docs/writing/drafts/YYYY-MM-DD-<topic>/section-N-<title>.md`

文件头：
```markdown
---
section: N
title: [章节标题]
word_count: XXX
status: draft
date: YYYY-MM-DD
---
```

## 人工检查点

### 触发时机
- 完整模式：每完成2-3节
- 快速模式：完整草稿完成后

### 检查点呈现格式

```markdown
## ✅ 检查点：第N-M节完成

**已完成章节：**
| 章节 | 标题 | 字数 | 状态 |
|------|------|------|------|
| 第N节 | [标题] | XXX字 | ✅ |
| 第N+1节 | [标题] | XXX字 | ✅ |
| 第N+2节 | [标题] | XXX字 | ✅ |

**进度：** [已完成字数] / [目标字数]（[百分比]）

**自检摘要：** 各节要点覆盖完整，风格一致，衔接自然。

**接下来：** 第N+3节 [标题]

**你可以：**
1. ✅ **继续** — 进入下一批
2. ✏️ **修改** — 对已完成章节提出修改意见
3. ⏸️ **暂停** — 先停在这里，稍后继续
```

## 草稿合并

所有章节完成后：

1. 按顺序读取所有章节文件
2. 去除每节的元数据头
3. 合并为单一文档
4. 通读检查整体连贯性
5. 添加文章头部信息
6. 保存至 `docs/writing/YYYY-MM-DD-<topic>-draft.md`

**合并后的文档格式：**
```markdown
---
title: [文章标题]
date: YYYY-MM-DD
type: [文章类型]
status: 草稿
word_count: XXXX
---

# [文章标题]

[副标题或导语（如有）]

---

[合并后的全文内容]
```

## 合并后过渡

**"全文草稿已完成！**

**基本信息：**
- 总字数：XXXX（目标：XXXX）
- 章节数：N
- 文章类型：[类型]

**已保存到：** `docs/writing/YYYY-MM-DD-<topic>-draft.md`

**接下来进入审校环节，我会从内容、表达、规范三个维度检查全文。继续吗？"**

**用户确认后 →** 调用 `story-workshop:文章审校`

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 某节素材不足 | 标注"[待补充：需要XX信息]"，继续写其他部分 |
| 字数严重超标（>20%） | 检查是否有冗余内容，果断删减——内网文章默认约 2000 字，读者没耐心读更长的 |
| 全文接近 2500 字 | 立即停下来审视，是否有可以砍掉的段落或合并的章节 |
| 风格突然跑偏 | 回读写作简报中的风格要求，重新校准 |
| 上下文衔接困难 | 回顾写作计划中的衔接设计，必要时调整章节顺序 |
| 用户在检查点要求大改 | 根据反馈修改已完成章节，必要时回到规划阶段调整 |

## 核心原则

- **忠于素材：** 不编造事实、不夸大数据、不虚构对话
- **让素材说话：** 好的原始素材（金句、数据、细节）胜过你的改写
- **节奏感：** 快板块和慢板块交替，不要全文一个速度
- **克制：** 宁可少写一句废话，也不要多写一段正确的废话
- **完成优先：** 先出完整草稿，再在审校阶段打磨。不要在写作阶段过度追求完美
